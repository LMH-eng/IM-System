# IM即时通讯系统

基于Qt和原生WinSocket开发的即时通讯软件，实现用户注册登录、好友管理、实时聊天、心跳保活、离线消息等核心功能。

## 功能特性

- 用户注册与登录
- 好友添加与管理
- 实时聊天（支持富文本）
- TCP长连接心跳保活
- 断线重连机制
- 离线消息存储与推送
- 离线好友申请
- 用户在线状态显示
- 好友签名动态更新

## 技术栈

| 层级 | 技术 |
|------|------|
| 客户端 | Qt 5、C++、QTcpSocket |
| 服务端 | C++、WinSocket、MySQL |
| 架构模式 | 中介者模式、分层架构 |

## 核心技术点

### 1. TCP粘包解决

采用**长度前缀法**：发送数据前先发送4字节长度，接收方先读长度，再循环接收完整数据。

```
[4字节长度][数据内容]
```

### 2. 心跳保活机制

基于**QTimer定时器**实现TCP长连接心跳保活：
- 客户端每30秒发送轻量级心跳包
- 服务端设置90秒超时阈值检测连接状态
- 自动识别并清理异常断开的连接
- 解决TCP半开连接导致的资源泄漏问题

### 3. 中介者模式

网络层与业务层通过中介者交互，降低耦合度：
- 修改网络实现（TCP换UDP）只需改中介者
- 业务层无需关心网络细节

```
┌─────────┐     ┌─────────┐     ┌─────────┐
│   UI层   │ ←→ │  中介者  │ ←→ │  网络层  │
└─────────┘     └─────────┘     └─────────┘
```

### 4. 信号槽跨类传递

Qt信号槽实现UI层与业务层通信，支持跨类信号转发。

### 5. 离线消息

用户离线时消息存入数据库，上线后自动推送。

### 6. 编码转换

客户端UTF-8，服务端GBK环境，实现双向编码转换，解决中文乱码问题。

## 项目结构

### 客户端

```
Client/
├── main.cpp              # 程序入口
├── kernel.h/cpp          # 核心业务逻辑
├── mainwidget.h/cpp      # 主窗口
├── logindia.h/cpp        # 登录窗口
├── frienditem.h/cpp      # 好友列表项控件
├── chatdia.h/cpp         # 聊天窗口
├── net/                  # 网络模块
│   ├── TcpClient.h/cpp   # TCP客户端封装
│   └── def.h             # 协议定义
└── mediator/             # 中介者模块
    └── TcpClientMediator.h/cpp
```

### 服务端

```
Server/
├── main.cpp              # 服务端入口
├── kernel.h/cpp          # 核心业务处理
├── net/                  # 网络模块
│   ├── TcpServer.h/cpp   # TCP服务端封装
│   └── def.h             # 协议定义
├── mediator/             # 中介者模块
└── MySQL/                # 数据库模块
```

## 运行方式

### 客户端
1. 使用Qt Creator打开 `Client/IMCllient.pro`
2. 编译运行

### 服务端
1. 配置MySQL数据库，执行建表SQL
2. 使用Visual Studio编译运行服务端
